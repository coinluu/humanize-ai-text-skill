# No Fabrication Rules

## 规则说明

自然化改写只能重组和优化用户提供的信息，不能新增看似真实但没有来源的事实。所有增强表达都必须建立在原文或用户补充材料之上。

## 适用场景

- 所有改写任务。
- 尤其适用于销售文案、简历、医疗健康、法律、金融、教育培训、本地生活和参考样本改写。

## 禁止新增

- 真实人物
- 真实客户案例
- 具体数据
- 收入结果
- 平台政策
- 法律结论
- 医疗结论
- 金融收益
- 个人经历
- 权威背书
- 机构认证
- 奖项排名
- 使用体验
- 产品效果
- 用户评价

## 允许内容

- 基于用户已提供信息重组表达。
- 使用明确标识的假设性场景。
- 使用"比如"、"假设"、"常见情况是"等提示，让读者知道不是事实案例。
- 把抽象表述改得更清楚，但不增加新事实。

## 假设性场景规则

可以写：

```text
比如，如果一个团队的问题是信息分散，那么这套流程可以帮助他们先把任务和责任人对齐。
```

不可以写：

```text
我们曾帮助一个团队把沟通效率提升了 50%。
```

除非用户提供该真实案例和数据。

## 禁止事项

- 禁止把"可能"改成"一定"。
- 禁止把"帮助"改成"保证"。
- 禁止把"适合"改成"最佳"。
- 禁止把假设写成亲历。
- 禁止从参考样本中借用事实补足用户原文。

## 示例

Before:

```text
这款工具可以帮助团队整理项目进度。
```

Bad After:

```text
已有 300 多家企业使用后明显提升了项目交付效率。
```

Correct After:

```text
这款工具的重点，是把项目进度、责任人和待处理事项放到同一个地方，减少来回确认的成本。
```
