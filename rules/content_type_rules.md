# Content Type Rules

## 规则说明

根据内容类型确定改写重点、强度和注意事项。内容类型优先于单纯平台语气；同一平台上的文章、标题、评论回复应采用不同处理方式。

## 适用场景

- 用户没有明确说明内容类型，需要 Agent 自动判断。
- 用户要求把文本改成文章、笔记、口播、销售文案、邮件、简历等指定形态。
- 同一文本需要在不同内容类型之间转换。

## 禁止事项

- 禁止把一种内容类型的语气硬套到另一种类型上。
- 禁止把商务、简历、法律、医疗等严肃内容改成过度社交化表达。
- 禁止为了适配类型新增事实、案例、数据或经历。
- 禁止忽略内容类型中的必要结构，例如邮件的行动项、简历的真实职责、商务文档的条件和结论。

## article

判断特征：

- 多段落、观点展开、解释或论证。

改写重点：

- 优化开头、段落推进、观点层次和结尾。

改写强度建议：

- Level 2-3；高风险内容 Level 1-2。

注意事项：

- 不新增案例和数据。

## social_note

判断特征：

- 社交平台笔记、经验分享、种草说明、生活方式内容。

改写重点：

- 提升扫读感、具体感和自然表达。

改写强度建议：

- Level 2-3；样本引导可 Level 4。

注意事项：

- 不伪造亲身体验。

## short_video_script

判断特征：

- 用于口播、短视频、直播或音频表达。

改写重点：

- 句子短、停顿自然、逻辑好听懂。

改写强度建议：

- Level 2-3。

注意事项：

- 不为抓注意力夸大事实。

## sales_copy

判断特征：

- 推广产品、服务、课程、活动、咨询或方案。

改写重点：

- 明确目标用户、价值、证据和行动。

改写强度建议：

- Level 2-3；高风险或收益类 Level 1-2。

注意事项：

- 不承诺结果，不编造客户案例。

## title

判断特征：

- 标题、主题行、视频标题、文章题目。

改写重点：

- 清楚、准确、有吸引力。

改写强度建议：

- Level 2-5；高风险标题 Level 1-2。

注意事项：

- 不使用误导性悬念或虚假强结论。

## caption

判断特征：

- 图片、视频或社交内容配文。

改写重点：

- 简短、贴合场景、有情绪但不过火。

改写强度建议：

- Level 1-3。

注意事项：

- 不强行升华。

## comment_reply

判断特征：

- 评论、评价、质疑、咨询或互动回复。

改写重点：

- 回应具体问题，语气自然，有边界。

改写强度建议：

- Level 1-2。

注意事项：

- 不激化冲突，不承认未确认责任。

## private_message

判断特征：

- 私信、社群私聊、跟进、邀约或提醒。

改写重点：

- 意图清楚，语气自然，不压迫。

改写强度建议：

- Level 1-2。

注意事项：

- 不强行熟络，不制造虚假关系。

## email

判断特征：

- 邮件正文、主题、跟进、道歉、申请或沟通。

改写重点：

- 结构清楚、礼貌、行动明确。

改写强度建议：

- Level 1-2。

注意事项：

- 保留关键条件、日期、责任和请求。

## resume

判断特征：

- 简历、求职信、项目经历、个人总结。

改写重点：

- 突出真实职责、动作和结果。

改写强度建议：

- Level 1-2。

注意事项：

- 不编造职位、数据、项目、结果或奖项。

## business_doc

判断特征：

- 汇报、方案、纪要、计划、制度、SOP。

改写重点：

- 结论、依据、风险、下一步清晰。

改写强度建议：

- Level 1-2。

注意事项：

- 保留专业术语和约束条件。

## knowledge_explanation

判断特征：

- 概念解释、教程、科普、方法说明。

改写重点：

- 降低理解门槛，保持准确。

改写强度建议：

- Level 2；高风险知识 Level 1。

注意事项：

- 例子必须来自用户或明确标识为假设。

## generic_text

判断特征：

- 无明确内容类型，或仅需一般自然化。

改写重点：

- 保留原意，提高自然度和可读性。

改写强度建议：

- Level 2。

注意事项：

- 不为了丰富而添加新信息。

## 示例

Source:

```text
我们需要提升沟通效率，减少项目推进中的信息差。
```

As `business_doc`:

```text
建议优化项目沟通流程，重点减少任务分配、进度同步和问题反馈中的信息差，提升协作效率。
```

As `short_video_script`:

```text
项目推进慢，很多时候不是大家不努力，而是信息没对齐。谁负责、进展到哪、卡在哪里，如果这些都不清楚，沟通成本就会越来越高。
```
