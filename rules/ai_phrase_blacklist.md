# AI 味表达黑名单

## 规则说明

本文件用于识别常见中文 AI 味、模板感、空泛化表达，并指导 Agent 结合语境重写。黑名单不是机械替换表；同一个词在不同场景中可能合理，也可能显得空泛。

## 适用场景

- 用户要求去 AI 味、自然化、像真人写。
- 文本存在宏大开场、空泛总结、套话过多、营销腔或官方腔。
- 需要把 AI 初稿改成公众号、小红书、口播、商务文档等可发布文本。

## 执行原则

- 先判断该表达是否为空泛套话。
- 如果表达承载了真实含义，保留含义并换成具体说法。
- 如果表达没有信息量，直接删除。
- 不做逐词替换；优先改写整句或整段。
- 替代表达必须来自原文信息，不得新增事实。

## 常见黑名单与替代表达原则

| AI 味表达 | 常见问题 | 更自然的处理原则 |
| --- | --- | --- |
| 在当今时代 | 宏大但无具体场景 | 直接进入具体问题、对象或读者处境 |
| 随着……快速发展 | 套路式背景 | 改成具体变化，例如"最近用户更在意..." |
| 值得注意的是 | 连接词堆叠 | 直接说重点，或用"但这里有一点要看清" |
| 总的来说 | 机械总结 | 如果无新信息则删除；需要收束时直接给结论 |
| 综上所述 | 论文式收尾 | 改成自然结论或行动建议 |
| 首先、其次、最后 | 模板三段式 | 改成按重要性、场景或行动顺序组织 |
| 不仅……而且…… | 过度对称 | 拆成自然句，突出真正重点 |
| 赋能 | 抽象、商业套话 | 说清"帮助谁做成什么" |
| 打造闭环 | 空泛管理话术 | 说清流程如何衔接、哪里减少断点 |
| 多维度 | 信息不具体 | 写出具体维度，或删除 |
| 深度解析 | 标题腔 | 改成"讲清楚..."、"拆开看..." |
| 提升效率 | 常见但空泛 | 写清提升哪一步效率 |
| 解决方案 | 泛商业词 | 改成"做法"、"方案"、"处理办法" |
| 全面升级 | 夸张但不具体 | 写清升级了什么 |
| 释放潜能 | 空泛激励语 | 改成具体能力或结果，但不得夸大 |
| 重要抓手 | 官方腔 | 改成"关键动作"、"主要办法" |
| 底层逻辑 | 滥用概念词 | 改成"核心原因"、"关键关系" |
| 生态体系 | 宏大商业词 | 写清参与方、产品或环节 |
| 助力 | 泛化动词 | 改成"帮助"或直接写动作 |
| 高质量发展 | 官方口径 | 写成具体目标或质量标准 |
| 全方位 | 夸大覆盖 | 写清覆盖范围；不确定则删除 |
| 体系化 | 抽象 | 改成"按步骤"、"形成固定流程" |
| 标准化 | 可能有真实含义 | 若有标准则保留；无标准则改成"统一做法" |
| 智能化升级 | 科技套话 | 写清自动化、提醒、识别、生成等具体能力 |

## 禁止事项

- 禁止把黑名单词机械替换成另一个套话。
- 禁止为了自然化新增事实、案例或数据。
- 禁止把专业术语误删；先判断是否为行业必要表达。
- 禁止把严肃内容改得过度口语化。

## 示例

### 示例 1

改写前：

```text
在当今时代，企业需要通过数字化工具赋能业务，实现高质量发展。
```

改写后：

```text
企业现在更需要把流程里的重复工作交给工具处理，让团队把时间放回客户沟通和业务判断上。
```

### 示例 2

改写前：

```text
首先，我们要提升效率；其次，要打造闭环；最后，实现全面升级。
```

改写后：

```text
这件事的关键，是先把重复步骤固定下来，再让前后环节接得上，避免问题反复靠人工补救。
```
