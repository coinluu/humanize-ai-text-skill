# Douyin Script Example

## 场景

抖音口播稿。用户希望前三秒有钩子，句子短，适合朗读，有节奏。

## 输入信息

```yaml
mode: platform_humanize
platform: douyin
content_type: short_video_script
rewrite_intensity: Level 3
output_mode: standard
allow_hypothetical_scenarios: true
```

## Before

```text
很多企业在使用 AI 工具时，往往只关注单点工具的使用，而忽视了工作流的整体搭建，因此无法真正发挥 AI 对业务效率提升的作用。
```

## AI 感诊断

- 句子太长，不适合口播。
- "单点工具、整体搭建、发挥作用"偏书面和抽象。
- 开头没有前三秒钩子。
- 缺少停顿和节奏，听众不容易一次听懂。

## 改写策略

- 先用一句钩子指出常见误区。
- 把长句拆成短句，适合朗读。
- 用假设性场景解释"工具"和"工作流"的区别。
- 保留原意：问题不在工具，而在流程没有串起来。

## After

```text
你以为公司用了 AI，效率就一定会变高吗？

不一定。

很多团队的问题，不是不会用 AI 工具。

而是每个人都在单独用。

写文案用一个工具。
整理表格用一个工具。
回复客户又用另一个工具。

看起来很先进，但流程还是断的。

真正有用的 AI 工作流，是把这些动作接起来。

什么内容先让 AI 处理？
哪里必须人工判断？
结果要同步到哪里？
下一步由谁跟进？

这些问题没想清楚，工具越多，反而越乱。

所以，别急着追新工具。
先把你的工作流程画出来，再看哪些环节适合交给 AI。
```

## 修改说明

- 增加前三秒反问钩子。
- 将原文长句拆成多个短句，增强口播节奏。
- 使用假设性工作场景帮助听众理解，但没有虚构真实客户案例。
- 保留"单点工具不足、工作流更重要"的核心观点。

## 风险提示

未发现高风险行业内容。文中场景是通用假设，不代表真实客户结果。
