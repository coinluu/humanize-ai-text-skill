# Business Doc Example

## 场景

商务文档。要求自然但专业，克制、清楚、准确，不要过度口语化。

## 输入信息

```yaml
mode: basic_humanize
content_type: business_doc
rewrite_intensity: Level 1
output_mode: standard
allow_hypothetical_scenarios: false
```

## Before

```text
为了进一步提升团队协作效率，推动项目管理流程实现体系化、标准化和智能化升级，我们建议引入 AI 工作流工具，并通过持续优化机制，全面赋能项目执行过程。
```

## AI 感诊断

- "体系化、标准化、智能化升级、全面赋能"堆叠较多，商务含义不够清楚。
- 句子过长，核心建议不突出。
- "持续优化机制"没有说明具体要优化什么。
- 语气偏口号化，不够决策友好。

## 改写策略

- 保留正式语气，但压缩套话。
- 先明确建议，再说明目的。
- 将抽象表达改成项目管理中的具体环节。
- 不加入新的成本、收益或实施结果。

## After

```text
建议引入 AI 工作流工具，用于辅助项目管理中的任务整理、进度同步和问题跟进。

该工具的主要价值在于减少重复沟通，让团队更清楚地看到事项负责人、当前进展和待处理问题。后续可根据实际使用情况，逐步优化流程配置和协作规则。
```

## 修改说明

- 将长句拆成建议和说明两部分。
- 删除"全面赋能"等抽象表达。
- 把"项目执行过程"具体化为任务、进度和问题跟进。
- 保持商务文档的克制和准确，没有过度口语化。

## 风险提示

未发现高风险内容。改写未新增预算、效率提升比例、实施周期或确定性收益。
