# Anti-Overhumanize Example

## 场景

展示错误的过度真人化版本，以及正确的克制真人化版本。错误版本表现为过度口水、油腻、营销号；正确版本自然、克制、有判断。

## 输入信息

```yaml
mode: basic_humanize
content_type: business_doc
rewrite_intensity: Level 2
output_mode: standard
allow_hypothetical_scenarios: false
```

## Before

```text
本次流程优化将帮助团队减少重复沟通，提升任务协作效率，并进一步明确项目推进过程中的责任分工。
```

## AI 感诊断

- 原文表达清楚，但略像商务套话。
- "流程优化、提升效率、责任分工"偏抽象。
- 需要自然化，但仍应保持专业语气。

## 错误的过度真人化版本

```text
说真的，这次优化就是来救大家的！以后不用再来回扯皮，谁该干啥一眼就懂，效率直接起飞，团队协作也会舒服很多。
```

## 错误原因

- "救大家的、扯皮、直接起飞"不适合商务文档。
- 语气过度口水化和营销号化。
- "效率直接起飞"夸大效果。
- 改写改变了原文的克制程度。

## 改写策略

- 保留商务场景的专业语气。
- 将抽象词改得更具体，但不夸张。
- 使用更自然的句式，减少套话感。
- 不新增确定性结果承诺。

## After

```text
本次流程优化的重点，是减少团队在任务确认和进度同步上的重复沟通。

调整后，项目推进中的负责人、协作事项和后续动作会更清楚，团队也更容易判断每一项工作当前处在哪个阶段。
```

## 修改说明

- 把"提升任务协作效率"具体化为任务确认和进度同步。
- 保留"明确责任分工"的意思，改成负责人、协作事项和后续动作更清楚。
- 语气更自然，但没有变成口水化表达。
- 未加入额外结果、数据或情绪化承诺。

## 风险提示

未发现高风险行业内容。需注意商务文本不应为了真人感而牺牲专业性和准确性。
