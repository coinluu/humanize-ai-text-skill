# 小红书示例

## 场景

小红书 AI 工作流顾问内容。用户希望降低官方感，用短段落表达真实观点，适合发布为顾问类笔记。

## 输入信息

```yaml
mode: platform_humanize
platform: xiaohongshu
content_type: social_note
rewrite_intensity: Level 3
output_mode: standard
allow_hypothetical_scenarios: true
```

## 改写前

```text
AI 工作流顾问可以帮助企业构建智能化工作体系，通过多维度分析业务流程，打造高效协同的解决方案，从而提升组织效率，实现业务增长。
```

## AI 感诊断

- "智能化工作体系、多维度分析、解决方案"偏官方，缺少小红书笔记的真实感。
- 原文只说价值，没有说明适合什么人、解决什么具体问题。
- "实现业务增长"结果导向过强，但缺少依据。
- 句子过满，没有适合手机阅读的停顿。

## 改写策略

- 改成短段落，适合小红书扫读。
- 用"适合谁/不适合谁"增强判断和边界。
- 将"业务流程"落到常见假设场景，不写成真实客户结果。
- 删除"实现业务增长"这种无法验证的强结果表达。

## 改写后

```text
如果你已经开始用 AI，但团队里还是各用各的、流程很散，那 AI 工作流顾问更像是帮你把这些工具重新串起来的人。

它解决的不是"会不会用一个 AI 工具"，而是这些问题：

- 哪些重复工作可以交给 AI 先处理
- 哪些环节必须保留人工判断
- 提示词、表格、文档、审批怎么接起来
- 团队成员怎么用同一套流程协作

比较适合已经有明确业务流程、但执行里经常卡住的团队。

不太适合只想找一个万能工具、或者希望 AI 直接替你解决所有问题的人。AI 工作流真正有用的地方，还是把具体工作拆清楚，再一点点接上。
```

## 修改说明

- 把官方表述改成更具体的使用场景。
- 用短段落和列表提高小红书阅读友好度。
- 增加适合与不适合人群，增强可信度和边界感。
- 删除未经支持的"业务增长"承诺。

## 风险提示

未虚构具体客户、收入结果或真实案例。文中提到的团队场景属于假设性说明。
